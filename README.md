# MedVisor :  Medical Education Assistant


#What it does

Ask MedVisor any medical question (e.g. *"What is an ischemic stroke?"*) and it will:

1. **Retrieve** the most relevant facts from a curated medical knowledge base (semantic search with sentence embeddings).
2. **Generate** a clear, patient-friendly explanation grounded in those facts using an LLM (Retrieval-Augmented Generation).
3. **Illustrate** the concept with an educational image generated by Stable Diffusion (SDXL-Turbo).


All wrapped in an interactive **Gradio** web UI.

```
   │     (MiniLM + cosine) │                            │                                │

   └────────────────────────┘
```

**Technology stack**

| Layer | Technology |
|-------|-----------|
| UI | Gradio |
| LLM | OpenAI-compatible API (OpenRouter / Big Pickle / Ollama) |
| Retrieval | sentence-transformers (`all-MiniLM-L6-v2`) + cosine similarity |
| Image generation | Stable Diffusion (SDXL-Turbo via diffusers) |
| Knowledge base | Curated JSON of medical topics |

# Running Locally

### 1. Clone & install
```bash
git clone https://github.com/giovanni850730/medvisor.git
cd medvisor
pip install -r requirements.txt
```

### 2. Configure the LLM backend
Copy `.env.example` to `.env` and add your API key (OpenRouter is recommended; it has a free tier):

```bash
cp .env.example .env
# then edit .env and set LLM_API_KEY
```

Or export environment variables directly:
```bash
export LLM_BASE_URL="https://openrouter.ai/api/v1"
export LLM_API_KEY="sk-or-v1-..."
export LLM_MODEL="meta-llama/llama-3.1-8b-instruct"
```

### 3. Launch
```bash
python app.py
```
Open the local URL (or the public `share` link) printed in the terminal.

## Running on Google Colab

A ready-to-run notebook is provided: **`MedVisor_Colab.ipynb`**.

1. Open it in Colab and select a **GPU runtime** (`Runtime → Change runtime type → T4 GPU`).
2. Add your `LLM_API_KEY` in the Colab **Secrets** panel (🔑 icon) or in the config cell.
3. Run all cells. The last cell launches the app and prints a public Gradio link.

> A GPU is recommended so SDXL-Turbo can generate images quickly. The LLM runs through an API, so it does not consume local GPU memory.

## Configuration Options

| Variable | Default | Description |
|----------|---------|-------------|
| `LLM_BASE_URL` | `https://openrouter.ai/api/v1` | OpenAI-compatible endpoint |
| `LLM_API_KEY` | — | API key for the LLM provider |
| `LLM_MODEL` | `meta-llama/llama-3.1-8b-instruct` | Model id |
| `IMAGE_MODEL` | `sdxl-turbo` | `sdxl-turbo` (fast) or `sd15` |
| `KB_PATH` | `knowledge_base/medical_facts.json` | Knowledge base file |
